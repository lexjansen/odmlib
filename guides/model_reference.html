Model Reference

odmlib provides Python model packages for each supported CDISC standard. Select the appropriate package based on the document type you are working with.

ODM 1.3.2 Model Structure

The ODM 1.3.2 model follows the CDISC ODM specification hierarchy:

ODM
└── Study
    ├── GlobalVariables (StudyName, StudyDescription, ProtocolName)
    ├── BasicDefinitions (MeasurementUnit)
    └── MetaDataVersion
        ├── Protocol
        │   └── StudyEventRef (links to StudyEventDef)
        ├── StudyEventDef
        │   └── FormRef (links to FormDef)
        ├── FormDef
        │   └── ItemGroupRef (links to ItemGroupDef)
        ├── ItemGroupDef
        │   └── ItemRef (links to ItemDef)
        ├── ItemDef
        │   └── CodeListRef (links to CodeList)
        ├── CodeList (CodeListItem / EnumeratedItem)
        ├── ConditionDef
        ├── MethodDef
        └── Presentation

ODM 2.0 Key Differences

ODM 2.0 uses a flatter structure and adds workflow support:

  • StudyEventDef directly contains ItemGroupRef (no FormDef/FormRef layer)

  • New WorkflowDef and WorkflowRef elements

  • Study has StudyName and ProtocolName as string attributes (not child elements)

  • Timing elements: AbsoluteTimingConstraint, RelativeTimingConstraint, etc.

ODM 2.0 Known Limitations

The ODM 2.0 model is a draft. v0.2.0 aligned it with the ODM 2.0 XSD for the core CRF/dataset metadata subset (TranslatedText/@Type is now required with a builder default, duplicate Arm/CheckValue removed, 12 missing value-set keys added, permissive mode can bypass an unregistered value set). Five structural features remain deferred to v0.2.1 and produce schema-invalid output if used under model_package="odm_2_0":

  • ConditionDef lacks the XSD-required MethodSignature child (and required Description) – add_condition_def() is unsafe for ODM 2.0.

  • FormalExpression is text-based; the XSD is element-based (Code | ExternalCodeLib) – do not pass formal_expression to add_method_def() for ODM 2.0.

  • Protocol carries StudyEventRef, removed in the ODM 2.0 schema – add_study_event_ref() is unsafe for ODM 2.0.

  • MetaDataVersion.StudyTiming placement (the XSD puts timing under Protocol/StudyTimings).

  • StudyEventGroupDef cannot satisfy its required (StudyEventGroupRef?, StudyEventRef?) child group.

The ItemDef attribute-set alignment (drop FractionDigits / DatasetVarName / SDSVarName, add DisplayFormat / VariableSet) landed in v0.2.1; its XSD ItemDef/ValueListRef child element remains deferred.

See the ROADMAP “v0.2.1 – ODM v2.0 Model/XSD Alignment” section and ODM20-MODEL-XSD-DIFFERENCES_PLAN.md for full detail and the schema-valid safe-subset workaround.

Define-XML Extensions

Define-XML extends ODM 1.3.2 with regulatory submission metadata:

New elements in Define-XML 2.1:

  • ValueListDef – permissible value lists

  • WhereClauseDef – filtering conditions

  • CommentDef – annotation definitions

  • Standards / Standard – CDISC standard references (e.g., SDTM, ADaM)

  • leaf – external document references

Extended elements (additional attributes):

  • ItemGroupDef – adds Structure, ArchiveLocationID, CommentOID, StandardOID, HasNoData

  • ItemDef – adds DisplayFormat, CommentOID, Origin, ValueListRef

  • MetaDataVersion – adds DefineVersion, Standards, ValueListDef, WhereClauseDef, CommentDef

Selecting the Right Loader

import odmlib.loader as LD
import odmlib.odm_loader as OL
import odmlib.define_loader as DL

# ODM 1.3.2 XML
loader = LD.ODMLoader(OL.XMLODMLoader(model_package="odm_1_3_2"))

# ODM 2.0 XML
loader = LD.ODMLoader(OL.XMLODMLoader(model_package="odm_2_0"))

# Define-XML 2.1 XML
loader = LD.ODMLoader(DL.XMLDefineLoader(model_package="define_2_1"))

# Define-XML 2.0 JSON
loader = LD.ODMLoader(DL.JSONDefineLoader(model_package="define_2_0"))

# Dataset-XML 1.0.1
loader = LD.ODMLoader(OL.XMLODMLoader(model_package="dataset_1_0_1"))

Creating Custom Extensions

To create a proprietary ODM extension:

  1. Create a new model module importing the base model

  2. Subclass existing elements to add new attributes/elements

  3. Register any new namespaces

  4. Use local_model=True with the full module path

# myext/model.py
import odmlib.odm_1_3_2.model as ODM
import odmlib.typed as T
import odmlib.ns_registry as NS

NS.NamespaceRegistry(prefix="myext", uri="http://example.com/myext")

class ItemDef(ODM.ItemDef):
    CustomAttr = T.String(namespace="myext")

# Load using local_model
loader = LD.ODMLoader(OL.XMLODMLoader(
    model_package="myext",
    local_model=True
))
