## Types of Contributions
We recognize all forms of contribution, including:
- 💻 Code
- 🏗️ Model
- 📖 Documentation
- 🐛 Bug reports
- 💡 Ideas and feedback
- 🔬 Domain expertise (CDISC, data exchange standards, foundational standards)
- 🧪 Testing and validation
- 🤝 Community support (reviews, discussions, onboarding)

## What Counts as a Contribution

A contribution is any input that materially improves the library, its correctness, usability, or alignment with CDISC 
standards and supported use cases.

We recognize contributions that meaningfully improve the project, including:
- Code changes
- Model changes or additions
- Documentation improvements
- Bug reports with actionable detail
- Feature suggestions or design input
- Domain expertise that influences direction

General expressions of support (e.g., "thanks!", "great project") are appreciated but are not typically listed here.
