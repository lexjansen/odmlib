# CONTRIBUTING

## Overview

Thank you for your interest in contributing to **odmlib**.

odmlib is an open-source library focused on working with ODM and Define-XML in the context of CDISC standards. 
Contributions are welcome from both **software developers** and **domain experts**.

Because odmlib operates in a standards-driven and regulated space correctness and standards alignment are important.

---

## Contribution Rubric

A contribution is considered meaningful if it:

* Improves functionality
* Enhances usability
* Ensures correctness
* Strengthens alignment with CDISC standards
* Contributes knowledge to the community

In general:

> If your contribution **changes the behavior, correctness, or direction of the project**, it qualifies.

The following typically do **not** qualify for inclusion in `CONTRIBUTORS.md`:

* General praise (“thanks”, “great project”)
* Non-actionable comments that do not fit into the above Ways to Contribute categories
* Emoji reactions

(These are still appreciated and encouraged.)

---

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add or update tests where appropriate
5. Open a pull request

---

## Pull Request Guidelines

### General Expectations

* Keep changes focused and scoped
* Include tests for new functionality
* Update documentation where needed
* Write clear commit messages
* Test data files appreciated

---

## Review Process

odmlib uses a governance model with two key roles:

* Maintainers
* Reviewers

### Minor Changes

* Require approval from at least one Reviewer

### Major Changes

* Require discussion and alignment with Maintainers

---

## Coding Standards

* Follow existing project structure and patterns
* Write clear, readable Python code
* Prefer explicitness over cleverness
* Maintain backward compatibility unless explicitly discussed

---

## Testing Expectations

* Add tests for all new functionality
* Ensure existing tests pass
* Include edge cases when relevant
* Prefer realistic data scenarios where possible

---

## Documentation Expectations

* Update documentation alongside code changes
* Provide examples for new features
* Clarify complex workflows (especially around CDISC usage)

---

## Contributor Recognition

Contributors who make meaningful contributions (per the rubric above) may be listed in `CONTRIBUTORS.md`.

* Contributors are listed by GitHub username by default
* Real names and affiliations may be included if voluntarily provided

If you would like your name added, updated, or removed, please open an issue or pull request.

---

## Communication

* Use GitHub Issues for bugs and feature requests
* Use Pull Requests for proposed changes
* Be respectful and constructive in all discussions

---

## Path to Greater Involvement

Contributors may take on larger roles over time:

### Reviewer

* Consistent, high-quality contributions
* Strong understanding of the codebase
* CDISC standards knowledge

### Maintainer

* Sustained leadership and high-impact contributions
* CDISC standards expertise

---

## Final Notes

odmlib aims to be:

* Technically robust
* Standards-compliant
* Useful in real-world clinical workflows
* Trusted library for AI generated code

Your contributions—whether code, domain expertise, or validation—are essential to achieving that goal.

Thank you for contributing!
